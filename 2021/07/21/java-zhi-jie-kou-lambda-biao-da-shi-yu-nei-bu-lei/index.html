接口、lambda表达式与内部类

接口

接口的概念：

对于某一个类，如果它具有一个接口，接口中含有多组要求，希望这个类能够实现这些功能。

1.疑问：
为什么不在类中点明某一个具体的方法，而是将方法的声明放在接口中？
1）强类型语言，在执行过程中如果有确定的接口与方法进行对接，不会出bug；

2）抽象类进行一次继承获得扩展类后不能继续继承（java拒绝多重继承）

2.接口实现：

public interface Comparable<T>{
	int compareTo(T other);//类似于模板
}

3.具体实现步骤：
1）将类声明为给定的接口：
例：class Employee implements Comparable

2）对接口中的所有方法实现定义：
例：public int compareTo(Employee other) return Double.compare(salary,other.salary);

4.注意点：

1）注意多态的使用，类似于精度损失，注意强制转换；

2）满足对称性、传递性和自反性，类相等判断依据，返回结果应当与equals相同；

3）技巧：在超类中设定接口对应算法供派生类应用；

接口属性与默认方法

1.接口变量的使用：
1）用instanceof检验一个对象是否实现了某个特定的接口；

2）引用实现了这个接口的类的对象；

例：

if(anObject instanceof Comparable){}

Comparable e = new Employee();

2.接口扩展：类似于继承

3.默认方法的使用：
1）是什么？

接口方法的默认实现，类似于c++中构建虚函数进行实现，在子类中同时可以实现覆盖操作；

2）为什么？

将迭代器设置为只读，从而使防止栈中没有数据不能正常退出；

“接口演化”：长期存在的接口增加新的方法会导致源代码兼容出现问题（如果新方法没有进行默认处理）

3）怎么做：
在接口内部：default boolean isEmpty(T other){return size==0;}

4）默认方法出现冲突（二义性）

各类超类优先，接口方法覆盖；

注意：

1）接口中可以声明静态常量；

2）通过类优先的原则可知，object类的方法不能通过默认方法进行无效处理；

其他：

1）Object类中的clone函数：生成对应类对象的一个精准的副本（实现cloneable接口）

2）Path接口：URI或字符串构建资源的路径

实现方法举例：

public static Path of(URI uri){…}

回调

用户采取特定的动作，程序根据特定的动作进行特定输出；

直接上代码：（打点计时器（雾））

package inherience;

import javax.swing.*;
import java.awt.*;
import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;
import java.time.Instant;

public class TimerTest {
    public static void main(String[] args) {
        var listener = new TimePrinter();//引入接口
        var timer = new Timer(1000,listener);//时间构造器，计时，1000为时间间隔
        timer.start();

}
class TimePrinter implements ActionListener{
    public void actionPerformed(ActionEvent event){
        System.out.println("At the tone,the time is " + Instant.ofEpochMilli(event.getWhen()));//输出返回后的时间
        Toolkit.getDefaultToolkit().beep();
    }
}

2.clone重载：（使用较少，了解即可）
1）区分默认clone函数（浅拷贝，拉），深拷贝：重写clone函数，引入Cloneable接口（为标记接口，只能用于查询时的instanceof）；

2）浅拷贝的缺点：
拷贝后副本类中的子类会随着样本类的变化而变化，即共享子类；

默认拷贝函数为受保护类型，只能拷贝本身，用途不大；

3）主要代码实现：

package inherience;

public class CloneTest {
    public static void main(String[] args) {
        var original = new Employee(...);
        original.setHireDay(...);
        Employee copy = original.clone();
        copy.raiseSalary(10);
        copy.setHireDay(2002,12,31);
    }
}
public class Employee{
    public Employee clone(){
        Employee cloned = (Employee) super.clone();//从基类开始生成派生类
        cloned.hireDay=(Date)hireDay.clone();
        return cloned;
    }
}

lambda表达式

原因

传递代码块，相较于传递带有特定方法的类，效率较高。

语法调用

1.注意：
无需指定lambda表达式的返回类型，同时lambda表达式的实现可以通过写方法的形式进行；

2.代码实现：

//比较器实现：
Arrays.sort(plants,(first,second)->first.length()-second.length());
//动作监视器实现：
var timer = new Timer(1000,event->System.out.println("The time is "+new Date()));
timer.start;
//(参数)->具体实现

方法引用

1.定义：当lambda表达式只调用一个方法，没有其他操作，可以进行简化，进行方法引用。

2.示例：

public class Greeter {
    public void greet(ActionEvent event){
        System.out.println("Hello,the time is " + Instant.ofEpochMilli(event.getWhen()));
    }
}
class RepeatedGreeter extends Greeter{
    public void greet(ActionEvent event){
        var timer = new Timer(1000,super::greet);
        timer.start();
    }
}

构造器引用

示例：Stream stream = names.stream().map(Person::new);//为各个列表元素调用Person（String）构造器；

注意事项

1.组成部分：一个代码块、参数、自由变量的值；

2.引入的变量在表达式内和表达式外进行值的改变都是不安全的操作；

内部类

1.示例：

//实现打点计时器
package innerClass;

import javax.swing.*;
import java.awt.*;
import java.awt.event.ActionEvent;
import java.awt.event.ActionListener;
import java.time.Instant;

public class InnerClassTest {
    public static void main(String[] args) {
        var clock = new TalkingClock(1000,true);
        clock.start();
        JOptionPane.showMessageDialog(null,"Quit program?");
        System.exit(0);
    }
}
class TalkingClock{
    private final int interval;
    private final boolean beep;
    public TalkingClock(int interval,boolean beep){
        this.interval=interval;
        this.beep=beep;
    }
    public void start(){
        var listener = new TimePrinter();
        var timer = new Timer(interval,listener);
        timer.start();
    }
    //内部类实现
    public class TimePrinter implements ActionListener{
        public void actionPerformed(ActionEvent event){
            System.out.println("At the tone,the time is " + Instant.ofEpochMilli(event.getWhen()));
            if(beep){
                Toolkit.getDefaultToolkit().beep();
            }//在包内实现了隐藏
        }
    }
}

2.内部类的作用：
1）在同一个包中实现隐藏，类似于c++中的函数实现，不在包外显示，因此不需要；

2）内部类的对象会有一个隐式引用，指向这个实例化对象的外对象（outer）。

例：使用内部类构成链表时，不同链表中的元素都是各自链表的一个引用；

3）内部类可以引用外部类的私有成员（有降低安全性的风险），同时可以访问局部常量；

3.注意事项：

如何进行隐式引用？ 利用OuterClass.this表示外部类；

静态内部类

//从数组中挑选最大的元素（只有double类型）
package staticInnerClass;

import java.util.ArrayList;

public class StaticInnerClassTest {
    public static void main(String[] args) {
        var values = new double[20];
        for(int i=0;i<values.length;i++)
            values[i]=100*Math.random();
        ArrayAlg.Pair p = ArrayAlg.minmax(values);
        System.out.println("min= " + p.getFirst());
        System.out.println("max= " + p.getSecond());
    }
}
class ArrayAlg{
    public static class Pair {
        private double first;
        private double second;

        public Pair(double f, double s) {
            first = f;
            second = s;
        }

        public double getFirst() {
            return first;
        }

        public double getSecond() {
            return second;
        }
    }
    public static Pair minmax(double[] values){
        double min = Double.POSITIVE_INFINITY;
        double max = Double.NEGATIVE_INFINITY;
        for (double v: values) {
            if(min>v) min=v;
            if(max<v) max=v;
        }
        return new Pair(min,max);
    }
}

代理

1.利用代理可以在运行过程中创建一组给定接口的新类，在程序运行的过程中进行创建；

2.代理类包含的方法：

指定接口所需方法以及Object类中的所有方法；

3.建立代理类需要的参数：

类加速器、Class对象数组、调用处理器；

4.代码示例：

package proxy;

import java.lang.reflect.InvocationHandler;
import java.util.Arrays;
import java.util.Random;

public class ProxyTest {
    public static void main(String[] args) {
        var elements = new Object[1000];
        for(int i=0;i< elements.length;i++){
            Integer value = i+1;
            var handler = new TraceHandler(value);
            Object proxy = Proxy.newProoxyInstance(ClassLoader.getSystemClassLoader(),new Class[]{Comparable.class},handler);
            elements[i]=proxy;
        }
        Integer key = new Random().nextInt(elements.length)+1;
        int result = Arrays.binarySearch(elements,key);
        if(result>=0)
            System.out.println(elements[result]);
    }
}
class TraceHandler implements InvocationHandler{
    private Object target;
    public TraceHandler(Object t){
        target = t;
    }
    public Object invoke(Object proxy,Method m,Object[] args)throws Throwable{
        System.out.print(target);
        System.out.print("."+m.getName()+"(");
        if(args!=null){
            for(int i=0;i<args.length;i++){
                System.out.print(args[i]);
                if(i<args.length-1)
                    System.out.println(",");
            }
        }
        System.out.println(")");
        return m.invoke(target,args);
    }
}
